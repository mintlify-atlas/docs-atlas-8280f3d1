GitMCP transforms any GitHub repository or GitHub Pages site into a remote [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) server — instantly, with zero setup. Point your AI coding assistant at a GitMCP URL and it gains accurate, real-time access to that project's documentation and code. No more hallucinated APIs or outdated examples.

## Why GitMCP?

AI coding assistants are only as good as their context. Without access to up-to-date documentation, they guess — and they guess wrong. GitMCP solves this by acting as a live bridge between your AI tool and any GitHub project.

## Get started in three steps

<Steps>
  <Step title="Build your GitMCP URL">
    Replace the domain in your GitHub URL:

    | Source | GitMCP URL |
    |--------|-----------|
    | `github.com/owner/repo` | `gitmcp.io/owner/repo` |
    | `owner.github.io/repo` | `owner.gitmcp.io/repo` |
    | Any repository (dynamic) | `gitmcp.io/docs` |
  </Step>
  <Step title="Add it to your AI assistant">
    Paste the GitMCP URL as a new MCP server in your IDE or AI tool. See the [integration guides](/integrations/cursor) for exact configuration steps.
  </Step>
  <Step title="Start coding without hallucinations">
    Ask your AI assistant about the library. It will automatically call GitMCP to fetch accurate, current documentation before answering.
  </Step>
</Steps>
