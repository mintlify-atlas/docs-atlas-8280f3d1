## URL Conversion

GitMCP works by converting a GitHub URL into a GitMCP URL. Use the table below to find the right format for your use case:

| GitHub URL | GitMCP URL |
|---|---|
| `github.com/owner/repo` | `gitmcp.io/owner/repo` |
| `owner.github.io/repo` | `owner.gitmcp.io/repo` |
| Any repository (dynamic) | `gitmcp.io/docs` |

<Steps>
  <Step title="Build your GitMCP URL">
    Take any GitHub repository URL and swap the domain.

    **Example:** Convert `https://github.com/langchain-ai/langgraph` to:

    ```
    https://gitmcp.io/langchain-ai/langgraph
    ```

    Replace `langchain-ai` with the GitHub username or organization, and `langgraph` with the repository name.
  </Step>

  <Step title="Add to your AI assistant">
    Open your Cursor configuration file at `~/.cursor/mcp.json` and add the GitMCP server:

    ```json
    {
      "mcpServers": {
        "gitmcp": {
          "url": "https://gitmcp.io/langchain-ai/langgraph"
        }
      }
    }
    ```

    <Note>
      Restart Cursor after saving the configuration file for the changes to take effect.
    </Note>
  </Step>

  <Step title="Prompt your AI">
    Start a conversation in your AI assistant and ask something that requires the repository's documentation:

    > "Add memory to my LangGraph agent"

    The AI will call GitMCP to fetch the latest documentation and code from the repository before generating its response — no hallucinations, no stale context.
  </Step>

  <Step title="Verify it's working">
    Look for a tool call in the conversation. When GitMCP is active, your AI assistant will show a tool invocation such as `fetch_langgraph_documentation` or `search_langgraph_documentation` before it responds.

    If you see that tool call, GitMCP is connected and working correctly.
  </Step>
</Steps>

<Tip>
  Don't want to pick a specific repository upfront? Use the generic endpoint `https://gitmcp.io/docs` instead. Your AI assistant will determine which GitHub repository to access based on the context of each request, giving you flexibility across multiple projects in a single session.
</Tip>
