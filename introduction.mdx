## What is GitMCP?

AI coding assistants are powerful, but they are limited by their training data. When you ask your assistant about a library it has never seen — or a version released after its training cutoff — it guesses. Those guesses look plausible but are wrong: hallucinated function signatures, deprecated APIs, invented configuration options.

GitMCP solves this by acting as a live bridge between your AI tool and any GitHub project. It transforms any public GitHub repository or GitHub Pages site into a remote [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) server. Your AI assistant connects to that server and retrieves the real, current documentation before responding. No hallucinations. No stale examples.

<Info>
  MCP (Model Context Protocol) is an open standard that lets AI assistants request structured context from external sources. GitMCP speaks MCP natively, so no custom plugins or wrappers are needed.
</Info>

## Two server modes

GitMCP offers two modes depending on how you work.

**Specific repository** — point GitMCP at a single project. Use this when you work regularly with a particular library and want your assistant to always target that project. This is the recommended mode for most workflows.

| GitHub URL | GitMCP URL |
|---|---|
| `github.com/owner/repo` | `gitmcp.io/owner/repo` |
| `owner.github.io/repo` | `owner.gitmcp.io/repo` |

**Generic server** — use `gitmcp.io/docs` when you need to switch between many different repositories. The AI assistant determines which repository to query based on the context of your question. This mode is more flexible but requires the model to correctly identify the target project each time.

  <Card title="Latest documentation" icon="rotate">
